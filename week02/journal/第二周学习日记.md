# RESTFUL API

## 基础语法(选择动作)

- GET查询资源（注意登录算Post）
  - `GET /users` --查询所有用户
  - `GET /users/123` --查询id为123的用户
- PUT 完整更新资源
  - `PUT /users/123` --完整更新用户123
- PATCH 部分更新资源
  - `PATCH /users/123` --只更新123的部分字段，通常用于更精细的操作
- POST 创建资源以及**执行操作**
  - `POST /users` --创建新用户
  - `POST /users/login` -- 用户登录
- DELETE 删除资源
  - `DELETE /users/123` --删除用户123

## 添加查询条件

- 分页
  - `/users?page=2&limit=10`(查询第二页，每页十条信息)
- 过滤
  - `/users?gender=male&age=25`(查询性别为男，年龄25的用户)
- 排序
  - `/users?sort=created_at&order=desc`(按创建时间逆序排序用户)

# Controllers

首先@RestController，这样会以Json格式返回查找到的数据

然后@RequestMapping("/xxx"),全局返回路径的基础，例如在开头定义了这个之后再在下面定义一个getuser

```java
@Resource
UserMapper usermapper//顺便在这里说一下，这里@resource注解类似service的@Autowire，引入usermapper作为常驻接口

@GetMapping("/all")
public List<user> getUser(){
    return userMapper.findAll();
}
```

这段代码就会在网页`https//localhost:端口/user/all` 返回找到的所有用户数据的JSON数据

api接口就是类似这样实现

然后上面说的就是GET接口的实现，剩下的接口有

```
@PostMapping("/xxx")
@PutMapping("/xxx")
@DeleteMapping("/xxx")
```

大概就是不同的接口就是“@请求类型+Mapping”

然后对于**需要传参数**的接口，一般接收的也都是JSON数据，用@RequestBody解析成User文件

E.G

```
@PostMapping
public List<User> addUser(@RequestBody User user){
	return userMapper.insert(user);
}
```

# @RequestPart与@RequestBody区别

@RequestPart在有传文件时用

例如

```http
curl -X POST http://localhost:8080/upload \
  -F "file=@/path/to/your/document.pdf" \
  -F 'metadata={"name":"Alice","age":25,"department":"IT"};type=application/json'
```

对于这个请求，对应的api是

```java
@PostMapping(value = "/upload", consumes = MediaType.MULTIPART_FORM_DATA_VALUE)
public String uploadFile(
        @RequestPart("file") MultipartFile file,
        @RequestPart("metadata") UserMetadata metadata) {
    System.out.println("文件名: " + file.getOriginalFilename());
    System.out.println("元数据: " + metadata);
    return "上传成功";
}
```
